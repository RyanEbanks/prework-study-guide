Prework Study Guide

HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • The style element defines the css styles associated with a website.
  • The base element defines the base url for a webpage.
  • The meta element defines metadata such as keywords, descriptions etc for the website.
  • The a or anchor elemtn creates links to a webpage.
  • href points to that link.
  • Semantic HTML Elements are elements that describe themselves eg. body.
  • The img element contains information about images that are displayed on the webpage.
  • The src attribute defines the location of that image.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • A single change to a CSS rule will apply to all tagged page elements.
  • There are internal and external style sheets.
  • Declarations conatin two important components: The css property we want to apply and the value of the property.
  • The (*) Selector applies css styling to all elements in the HTML file.
  • You can combine 2 or more selectors into a single rule by using the comma (,). We use this if the elements have the same styling.
  • The img selector assigns property values to any image element in the markup.
  • The property display along with the block value assigns each image to start on a new line.

Git

  • git: must precede every git statement.
  • git status: checks what branch we are currently on.
  • git checkout: switches branches.
  • git checkout -b branch-name: creates a new branch and switches to it.
  • git add -A: adds all changes to the repository.
  • git commit -m: commits with a message, remove (-m) to normally commit.
  • git clone (add url): clones url.
  • git remote -v: verifies connection between clone and url.
